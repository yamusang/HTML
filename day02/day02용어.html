프론트엔드 용어 정리


  1. HTML
    • 마크업언어: 웹사이트의 모습을 기술하기 위한 언어
    • 문법오류에 관대
    • 동적인 화면 구성이 힘듬
    • DOM

  2. CSS
    • 스타일시트 언어:마크업 언어로 작성된 문서가 웹사이트에 표현되는 방법을 정해줌
    • 디자인적 요소를 HTML과 완전히 분리시킴
    • 문서의 해독 속도가 빨라짐
    • 동적인 기능 구현이 쉬워짐
    • 문서 크기가 작아짐
    • 디자이너와 프로그래머간의 협업이 용이해짐

  3. 자바스크립트
    • ECMA International의 프로토타입 기반의 프로그래밍 언어, 스크립트언어. 모든 웹 브라우저에 인터프리터가 내장되어 있음
    • 오늘날 HTML, CSS와 함께 웹을 구성하는 요소 중 하나
    • 부모 자식의 구분도 없으며, 메인 메소드 같은 것도 없고, 클래스-인스턴스 관계도 없으며, 모든 함수가 클래스도 될 수 있으며 인스턴스도 될 수 있다.
    • 멀티패러다임 언어로 명령형, 함수형, 객체 지향형 언어.

  4. HTTP
    • HyperTexT Protocol의 약자이다. 문서를 전송하기 위한 프로토콜.
    • 서버와 클라이언트의 사이에서 어떻게 메시지를 교환할지를 정해 놓은 규칙이다.
    • HTTP는 요청(Request)와 응답(Response)로 구성되어 있고, 클라이언트가 요청을 하면 서버가 응답을 하는 구조로 되어 있다.

  5. 웹사이트
    • 웹사이트(Website)는 HTTP 프로토콜을 써서 어떤 URL에서 받아오는 HTML로 쓰인 문서이다.
